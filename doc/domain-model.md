# SpecForge · Initial Domain Model

## Goal

Define the minimum governable core for phase 1 without dragging in entities from future complexity.

## Initial Aggregates

### `UserStory`

Represents the user story managed by SpecForge.

Responsibilities:

- stable identity
- reference to the primary artifact
- global state
- link to the applied workflow
- preservation of the initial source hash used to start the flow

### `WorkflowDefinition`

Describes the allowed flow.

Responsibilities:

- available phases
- valid transitions
- human-approval checkpoints
- allowed regression targets

### `WorkflowRun`

Executed instance of a workflow over a `UserStory`.

Responsibilities:

- current phase
- transition history
- global execution state

### `PhaseRun`

Execution of a concrete phase.

Responsibilities:

- input contract
- output artifacts
- result
- timestamps
- phase state

### `Artifact`

Artifact persisted in the repository.

Responsibilities:

- artifact type
- path
- logical version
- relation to phase and user story
- active or superseded marker

### `Checkpoint`

Explicit intervention point.

Responsibilities:

- reason
- approval state
- attached human instructions

### `WorkBranch`

Represents the work branch associated with a user story.

Responsibilities:

- user-selected base branch
- created branch name
- creation time
- link to the user story and active workflow

### `ReviewFinding`

Structured finding generated by review.

Responsibilities:

- severity
- detected non-compliance
- evidence
- target correction phase

## Deferred Entities

These stay outside the initial domain and should return only if implementation requires them:

- `ExecutionSlice`
- `ComponentScope`
- `DependencyEdge`
- `IssueLink`
- `ConversationContribution`
- `ManualEditDetection`
- `PauseRequest`
- `ProjectMetrics` as its own aggregate

## Initial Invariants

- A `UserStory` has a single active `WorkflowRun`.
- Only one `PhaseRun` in `running` state may exist per `WorkflowRun`.
- The next phase cannot start if the required checkpoint is not approved.
- Every regression must target a phase allowed by `WorkflowDefinition`.
- Every `ReviewFinding` must reference the affected phase or artifact.
- If the original source changes after `refinement` begins, the workflow does not incorporate it automatically.
- Every regeneration of an approved or already consumed output creates a new artifact version.
- A `completed` `UserStory` should not be reopened for substantial changes; the recommended path is to create a new user story.

## Minimum States

### Global State Of `UserStory`

- `draft`
- `active`
- `waiting_user`
- `paused`
- `blocked`
- `completed`

### State Of `PhaseRun`

- `pending`
- `ready`
- `running`
- `waiting_user`
- `completed`
- `failed`
- `regressed`

## Minimum Phase-1 Metrics

- `phaseLeadTime`
- `regressionCount`
- `manualInterventionCount`
- `reviewFailCount`
- `reviewPassCount`

These metrics must derive from observable events, not opaque inference.
