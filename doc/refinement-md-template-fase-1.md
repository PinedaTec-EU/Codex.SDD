# SpecForge · `01-spec.md` template phase 1

## Goal

Define the spec template as the consolidated functional artifact produced after `red-team` criticism and `blue-team` reconstruction.

## Principles

- it must contain the operational baseline spec of the user story
- it must make ambiguities and risks explicit
- it must include a `history log` at the top when the agent modifies it
- it must clearly expose what still needs human approval
